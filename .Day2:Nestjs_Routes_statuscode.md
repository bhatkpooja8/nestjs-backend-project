Route:
It is combination of path nd method

POST /users
In nestjs routes are defined in controller
HTTP METHODS:
GET
POSTPUT
DELETE

Route Params:
when u wanna pass some dynamic value through url , use @Param decorator
eg: GET /users/10

Query Params:
when u wanna pass some optional value through url , use @Query decorator
eg: GET /users?age=20

getUsers(@Query('age') age: number) {}

Request body:
when u wanna pass some data through body of request , use @Body decorator

{
  "name": "Pooja",
  "role": "Developer"
}

createUser(@Body() body: any) {}

HTTP Status Code:

Code	Meaning	     When to use

200	     OK	            Successful GET
201	    Created	         POST success
400	   Bad Request      Invalid input
404	     Not Found	    Data missing
500	    Server Error	Crash / bug

Provider:
A provider is a class managed by NestJS DI system

@Injectable()
export class UsersService {}

Provider scopes:
Singleton: default scope, one instance for entire app
Fast, memory efficient
@Injectable()
export class UsersService {}

Request Scope

New instance per request
@Injectable({ scope: Scope.REQUEST })
Real world ex: Gmail , used by all company but different information for each user

Transient Scope

New instance every injection
@Injectable({ scope: Scope.TRANSIENT })
